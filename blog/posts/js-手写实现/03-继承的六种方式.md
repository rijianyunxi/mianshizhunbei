# JavaScript 继承的六种方式：从小白到精通

## 一、继承的本质：原型链

在 JavaScript 中，**继承**的本质是通过原型链（prototype chain）实现的。每个构造函数都有一个 `prototype` 属性，指向一个对象；当用 `new` 关键字创建实例时，实例的内部属性 `[[Prototype]]`（即 `__proto__`）指向构造函数的 `prototype`。

当我们访问一个对象的属性时，JavaScript 引擎会沿着这条链向上查找，直到找到属性或到达链的末端（`Object.prototype`）。这就是**原型链查找机制**。

理解继承，就是理解如何构建这条链。

---

## 二、六种继承方式详解

### 1. 原型链继承

```javascript
function Parent() {
  this.name = '张三';
}
Parent.prototype.say = function() {
  console.log('Hello, I am ' + this.name);
};

function Child() {
  this.age = 18;
}
Child.prototype = new Parent(); // 关键：建立原型链

const child = new Child();
console.log(child.name); // '张三'
child.say();             // 'Hello, I am 张三'
```

**核心原理**：`Child.prototype = new Parent()` 将子类的原型对象替换为一个父类实例。由于父类实例的 `__proto__` 指向 `Parent.prototype`，整条原型链就形成了：`Child 实例 → Child.prototype (即Parent实例) → Parent.prototype → Object.prototype → null`。

**致命缺陷**：

- **引用类型共享**：父类的引用类型属性（如数组、对象）被所有子类实例共享。
```javascript
function Parent() {
  this.hobbies = ['coding', 'reading'];
}
function Child() {}
Child.prototype = new Parent();

const c1 = new Child();
const c2 = new Child();
c1.hobbies.push('gaming');
console.log(c2.hobbies); // ['coding', 'reading', 'gaming'] —— 灾难！
```

- **无法向父类构造函数传参**：创建子类实例时，无法给父类构造函数传递参数。

### 2. 构造函数继承（Constructor Stealing）

```javascript
function Parent(name) {
  this.name = name;
  this.hobbies = ['coding', 'reading'];
}
Parent.prototype.say = function() {
  console.log(this.name);
};

function Child(name, age) {
  Parent.call(this, name); // 关键：在子类构造函数中调用父类构造函数
  this.age = age;
}

const c1 = new Child('Alice', 18);
const c2 = new Child('Bob', 20);
c1.hobbies.push('gaming');
console.log(c1.hobbies); // ['coding', 'reading', 'gaming']
console.log(c2.hobbies); // ['coding', 'reading'] —— 隔离！
```

**核心原理**：通过 `Parent.call(this, ...args)` 在子类构造函数内部调用父类构造函数，借此将父类的实例属性复制到当前子类实例上。

**优点**：
- 每个子类实例都有独立的父类属性副本，引用类型不再共享
- 可以向父类传参

**缺陷**：
- **只能继承父类构造函数中的属性**，无法继承父类原型上定义的方法（`Parent.prototype.say` 在 `Child.prototype` 上根本访问不到）
- 每个实例都会创建一份父类方法的副本，浪费内存

### 3. 组合继承（Combination Inheritance）

```javascript
function Parent(name) {
  this.name = name;
  this.hobbies = ['coding', 'reading'];
}
Parent.prototype.say = function() {
  console.log(this.name);
};

function Child(name, age) {
  Parent.call(this, name); // 调用父构造函数（继承实例属性）
  this.age = age;
}
Child.prototype = new Parent(); // 建立原型链（继承方法）
Child.prototype.constructor = Child; // 修正 constructor 指向

const c1 = new Child('Alice', 18);
const c2 = new Child('Bob', 20);
c1.hobbies.push('gaming');
console.log(c1.hobbies); // ['coding', 'reading', 'gaming']
console.log(c2.hobbies); // ['coding', 'reading']
c1.say(); // 'Alice'
```

**核心原理**：结合原型链继承（继承方法）和构造函数继承（继承实例属性），取长补短。

**这是实际开发中最常用的方式**，解决了引用类型共享和传参问题。但它有一个隐藏的问题：

**性能问题**：父类构造函数被调用了两次——一次在 `new Parent()`（赋值给 `Child.prototype`），一次在 `Parent.call(this, name)`。这导致 `Child.prototype` 上会有一组父类的实例属性（但这些属性会被实例属性覆盖，不会被使用），浪费了内存。

### 4. 原型式继承（Prototypal Inheritance）

```javascript
function object(o) {
  function F() {}
  F.prototype = o;
  return new F();
}

const parent = {
  name: '张三',
  hobbies: ['coding', 'reading']
};

const child = object(parent);
child.name = '李四';
child.hobbies.push('gaming');

console.log(parent.name); // '张三' —— 基本类型隔离了
console.log(parent.hobbies); // ['coding', 'reading', 'gaming'] —— 引用类型共享！
```

**核心原理**：借助一个临时构造函数 F，将参数对象 o 作为 F 的原型，然后返回 F 的实例。ES5 的 `Object.create()` 就是这个原理的标准化实现。

**特点**：
- 不需要显式定义构造函数
- 适合创建"对象副本"（浅拷贝行为）
- 引用类型仍然共享（原型继承的本质）

### 5. 寄生式继承（Parasitic Inheritance）

```javascript
function createAnother(original) {
  const clone = Object.create(original); // 基于原对象创建一个副本
  clone.sayHi = function() {            // 增强：添加新方法
    console.log('Hi, I am ' + this.name);
  };
  return clone;
}

const parent = {
  name: 'Alice',
  hobbies: ['reading']
};

const child = createAnother(parent);
child.sayHi(); // 'Hi, I am Alice'
```

**核心原理**：在原型式继承的基础上，为克隆对象添加新方法或属性。命名中的"寄生"指的是在已有对象的基础上"长出"额外的功能。

**特点**：
- 可以在不改变原对象的情况下给副本添加功能
- 引用类型共享问题仍然存在
- 方法需要为每个实例重新创建，无法复用

### 6. 寄生组合继承（Parasitic Combination Inheritance）

这是 JavaScript 继承的**最优解**，也是面试中被问到"最优继承方式"时的标准答案：

```javascript
function Parent(name) {
  this.name = name;
  this.hobbies = ['coding', 'reading'];
}
Parent.prototype.say = function() {
  console.log(this.name);
};

function Child(name, age) {
  Parent.call(this, name); // 继承实例属性
  this.age = age;
}

// 关键函数：寄生组合继承的核心
function inheritPrototype(child, parent) {
  // 1. 创建新对象：parent.prototype 作为新对象的原型
  const prototype = Object.create(parent.prototype);

  // 2. 增强对象：修正 constructor 指向
  prototype.constructor = child;

  // 3. 赋值对象：替换子类的原型
  child.prototype = prototype;
}

inheritPrototype(Child, Parent);
```

**为什么是最优解？**

组合继承的问题是父类构造函数被调用了两次——一次在 `new Parent()`，一次在 `Parent.call(this)`。这导致 `Child.prototype` 上出现一组"多余"的父类实例属性。

寄生组合继承用 `Object.create(parent.prototype)` 创建了一个**中间对象**，这个中间对象的原型是 `parent.prototype`，自身不包含任何父类的实例属性。然后把这个中间对象赋值给 `child.prototype`。

这样一来：
- 父类构造函数**只调用了一次**（`Parent.call(this)`），不产生多余属性
- 原型链正确建立：子类实例 → Child.prototype → parent.prototype → Object.prototype
- `constructor` 指向正确
- 内存占用最小

这是 Doug Crockford 推荐的继承方式，也是绝大多数类库（如 TypeScript 编译 `class extends` 时生成的代码）所采用的底层实现。

## 三、ES6 Class extends 的底层原理

ES6 引入的 `class` 和 `extends` 关键字实际上是寄生组合继承的语法糖：

```javascript
class Parent {
  constructor(name) {
    this.name = name;
  }
  say() {
    console.log(this.name);
  }
}

class Child extends Parent {
  constructor(name, age) {
    super(name); // 等价于 Parent.call(this, name)
    this.age = age;
  }
}
```

`class extends` 背后的实现逻辑与寄生组合继承完全一致：子类原型继承自父类原型，同时在子类构造函数中调用 `super()`（相当于 `Parent.call(this)`）。

`super()` 必须**在访问 `this` 之前**调用，这是因为 ES6 的 class 语法不允许在调用 `super()` 之前使用 `this`。这是语法层面的强制要求，而不是设计缺陷。

## 四、常见面试问题

### 1. 组合继承最大的缺点是什么？

父类构造函数被调用了两次，导致子类原型上出现一组多余且无用的父类实例属性，造成内存浪费。虽然这些属性会被子类构造函数的实例属性覆盖，但它们确实存在于原型上，占用内存。

### 2. 寄生组合继承为什么是最优的？

它同时解决了三个问题：
- 正确建立原型链（方法可继承）
- 保留构造函数窃取（实例属性独立）
- 父类构造函数只调用一次（无多余属性）

从时间复杂度看，组合继承的 `new Parent()` 是 O(n)（创建父类全部实例属性），而寄生组合继承的 `Object.create()` 是 O(1)（只建立原型链接，不执行构造函数）。

### 3. instanceof 的原理是什么？

```javascript
function myInstanceof(left, right) {
  if (typeof left !== 'object' || left === null) return false;
  let proto = left.__proto__;
  const prototype = right.prototype;
  while (true) {
    if (proto === null) return false;
    if (proto === prototype) return true;
    proto = proto.__proto__;
  }
}
```

通过沿着对象的原型链向上查找，直到找到构造函数的 `prototype` 或到达链的末端。原理即如此。

### 4. 如何理解 `new` 关键字？

```javascript
function myNew(constructor, ...args) {
  let obj = {};                      // 1. 创建新对象
  obj.__proto__ = constructor.prototype; // 2. 链接原型
  let result = constructor.apply(obj, args); // 3. 绑定this并执行
  return (result !== null && (typeof result === 'object' || typeof result === 'function'))
    ? result : obj;                  // 4. 返回
}
```

`new` 的过程就是：创建对象 → 链接原型 → 绑定 this → 返回（或返回新对象）。

### 5. 原型链继承和构造函数继承的根本区别是什么？

原型链继承通过改变子类的原型（`Child.prototype = new Parent()`），让子类实例共享父类的属性——包括引用类型。构造函数继承通过 `Parent.call(this)` 在每个子类实例中复制父类属性，确保实例之间互不干扰。**原型链解决"方法共享"问题，构造函数解决"属性隔离"问题**。

## 五、总结对比

| 继承方式 | 实例属性 | 方法继承 | 引用类型 | 构造函数调用次数 | 评价 |
|---------|---------|---------|---------|---------------|------|
| 原型链继承 | 共享 | ✅ | ❌ 共享 | 0 | 引用类型共享，致命缺陷 |
| 构造函数继承 | 隔离 | ❌ | ✅ 隔离 | 1 | 无法继承方法 |
| 组合继承 | 隔离 | ✅ | ✅ 隔离 | 2（多余1次） | 较好，有多余属性 |
| 原型式继承 | 共享 | ✅ | ❌ 共享 | 0 | 引用类型共享 |
| 寄生式继承 | 共享 | ✅ | ❌ 共享 | 0 | 引用类型共享 |
| 寄生组合继承 | 隔离 | ✅ | ✅ 隔离 | 1 | ✅ 最优 |

寄生组合继承是 JavaScript 继承的最终答案，也是所有现代类库和框架（React 的 `class extends`、Vue 的 `extend`）背后的实现基础。
