# 闭包与作用域链

闭包是 JavaScript 最强大的特性之一，也是面试中的高频考点。本文深入剖析闭包的原理和应用场景。

## 1. 作用域与作用域链

### 作用域

作用域决定了变量的可访问范围：

```javascript
// 全局作用域
const global = 'global'

function outer() {
  // 函数作用域
  const outerVar = 'outer'
  
  function inner() {
    // 块级作用域
    const innerVar = 'inner'
    console.log(innerVar) // ✅ 内部可以访问
  }
  
  console.log(outerVar) // ✅
  // console.log(innerVar) // ❌ 外部访问不到
}

console.log(global) // ✅
```

### 作用域链

当访问一个变量时，JavaScript 会沿着作用域链向上查找：

```javascript
const a = 'global'

function outer() {
  const a = 'outer'
  
  function inner() {
    const a = 'inner'
    console.log(a) // 'inner' - 就近原则
    console.log(this.a) // 取决于调用方式
  }
  
  inner()
}
```

## 2. 什么是闭包？

### 闭包的定义

> 闭包是指函数能够访问其词法作用域之外的变量的能力。

```javascript
function outer() {
  const count = 0
  
  return function() {
    count++ // 访问了 outer 的变量
    return count
  }
}

const counter = outer()
console.log(counter()) // 1
console.log(counter()) // 2
console.log(counter()) // 3
```

## 3. 闭包的形成

```javascript
function createMultiplier(factor) {
  // factor 被内部函数引用，形成闭包
  return function(num) {
    return num * factor
  }
}

const double = createMultiplier(2)
const triple = createMultiplier(3)

console.log(double(5)) // 10
console.log(triple(5)) // 15
```

## 4. 闭包的应用场景

### 1. 数据私有化

```javascript
function createCounter() {
  let count = 0 // 私有变量
  
  return {
    increment() { count++ },
    decrement() { count-- },
    getCount() { return count }
  }
}

const counter = createCounter()
counter.increment()
counter.increment()
console.log(counter.getCount()) // 2
// counter.count // ❌ 访问不到
```

### 2. 函数柯里化

```javascript
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args)
    }
    return function(...args2) {
      return curried.apply(this, args.concat(args2))
    }
  }
}

const add = curry((a, b, c) => a + b + c)
console.log(add(1)(2)(3)) // 6
console.log(add(1, 2)(3)) // 6
```

### 3. 防抖和节流

```javascript
function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}
```

### 4. 缓存/记忆化

```javascript
function memoize(fn) {
  const cache = {}
  return function(...args) {
    const key = JSON.stringify(args)
    if (cache[key]) {
      return cache[key]
    }
    return cache[key] = fn.apply(this, args)
  }
}
```

## 5. 闭包与内存泄漏

### 常见问题

```javascript
function createLeak() {
  const largeData = new Array(1000000)
  
  // 不需要 largeData 但没有释放
  return function() {
    return 'I am leaking'
  }
}

const leaky = createLeak()
// largeData 不会被垃圾回收，因为它被内部函数引用
```

### 解决方案

```javascript
function createSafe() {
  const largeData = new Array(1000000)
  
  // 使用完大对象后清空
  const result = function() {
    return 'safe'
  }
  
  // 显式清空
  largeData = null
  
  return result
}
```

## 6. 面试高频问题

### Q1: 什么是闭包？

闭包是指函数能够记住并访问其词法作用域，即使函数在其词法作用域之外执行。

### Q2: 闭包的应用场景？

- 数据私有化
- 函数柯里化
- 防抖和节流
- 缓存/记忆化
- 模块模式

### Q3: 闭包会造成内存泄漏吗？

在某些情况下会：
- 闭包引用了大型对象但不再需要
- 闭包被全局变量持有，长期存在

解决：及时清空不需要的引用。

### Q4: for 循环中的闭包问题

```javascript
// ❌ 错误：所有函数都输出 5
for (var i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100)
}

// ✅ 解决方案 1: let
for (let i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100)
}

// ✅ 解决方案 2: 闭包
for (var i = 0; i < 5; i++) {
  (function(j) {
    setTimeout(() => console.log(j), 100)
  })(i)
}
```

## 7. 总结

闭包的核心：
1. 函数能访问定义时的作用域中的变量
2. 形成条件：内部函数引用外部函数的变量
3. 用途：数据私有、函数工厂、缓存等
4. 注意：可能导致内存泄漏
