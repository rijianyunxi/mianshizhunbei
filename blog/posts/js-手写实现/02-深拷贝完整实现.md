# 深拷贝完整实现：循环引用、类型处理与 WeakMap

## 一、为什么需要深拷贝

在 JavaScript 中，数据分为**值类型**和**引用类型**。值类型直接存储在栈中，赋值时是值的拷贝；引用类型（对象、数组）在栈中只存储指针，赋值时拷贝的是地址。这意味着：

```javascript
const a = { name: 'Alice', hobbies: ['reading', 'coding'] };
const b = a;        // b 和 a 指向同一个对象
b.name = 'Bob';     // 修改 b，a 也会变
b.hobbies.push('gaming'); // a 和 b 的 hobbies 都被改了

console.log(a.name); // 'Bob'
console.log(a.hobbies); // ['reading', 'coding', 'gaming']
```

如果你需要一个完全独立的对象副本——修改副本不会影响原对象——就需要**深拷贝**。对于简单对象，`JSON.parse(JSON.stringify())` 是一种常见的"半深拷贝"方案，但它的局限性非常多（后文详述）。

---

## 二、JSON.parse(JSON.stringify()) 的局限性

在动手实现深拷贝之前，先搞清楚为什么不能直接用它：

### 1. 无法拷贝函数（Function）

```javascript
const obj = {
  name: 'Alice',
  greet: function() { console.log('Hello!'); }
};

const cloned = JSON.parse(JSON.stringify(obj));
console.log(cloned.greet); // undefined
```

`JSON.stringify` 会自动忽略值为 `undefined`、函数、`Symbol` 的属性。

### 2. 无法拷贝 Date 对象

```javascript
const obj = { date: new Date() };
const cloned = JSON.parse(JSON.stringify(obj));
console.log(cloned.date); // "2026-03-28T00:00:00.000Z" —— 变成了字符串！
console.log(typeof cloned.date); // "string"
```

Date 对象被序列化为字符串后，无法反序列化为 Date 对象。

### 3. 无法拷贝正则表达式

```javascript
const obj = { pattern: /hello/gi };
const cloned = JSON.parse(JSON.stringify(obj));
console.log(cloned.pattern); // {} —— 空对象！
```

正则表达式和 Date 一样，JSON 没有对应的表示方式。

### 4. 无法处理循环引用

```javascript
const obj = { name: 'Alice' };
obj.self = obj; // 循环引用

JSON.stringify(obj); // TypeError: Converting circular structure to JSON
```

任何有循环引用的对象，`JSON.stringify` 都会抛出异常。

### 5. 无法拷贝 Map、Set、WeakMap、WeakSet

```javascript
const obj = { map: new Map([['key', 'value']]) };
const cloned = JSON.parse(JSON.stringify(obj));
console.log(cloned.map); // {} —— 空对象
```

### 6. 原型链丢失

如果对象继承自其他对象或类的原型，`JSON.stringify` 只拷贝自有属性，原型上的属性和方法全部丢失。

### 7. 特殊值问题

- `undefined` 属性被忽略
- `NaN` 和 `Infinity` 序列化为 `null`
- `Symbol` 键被忽略

因此，**在生产环境中，如果对象包含函数、Date、正则、循环引用等，`JSON.parse(JSON.stringify())` 基本上是不可用的**，必须手写深拷贝。

## 三、深拷贝的完整实现

下面是一个工业级深拷贝实现，覆盖了几乎所有边界情况：

```javascript
/**
 * 深拷贝完整实现
 * @param {*} obj 要拷贝的对象
 * @param {WeakMap} hash 解决循环引用的 WeakMap
 * @returns 深拷贝后的对象
 */
function deepClone(obj, hash = new WeakMap()) {
  // 处理原始类型：直接返回（因为原始类型已经是值拷贝）
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }

  // 处理循环引用：如果已经拷贝过，直接返回已有的引用
  if (hash.has(obj)) {
    return hash.get(obj);
  }

  // 处理 Date 对象
  if (obj instanceof Date) {
    return new Date(obj.getTime());
  }

  // 处理 RegExp 对象
  if (obj instanceof RegExp) {
    return new RegExp(obj.source, obj.flags);
  }

  // 处理 Map
  if (obj instanceof Map) {
    const cloneMap = new Map();
    hash.set(obj, cloneMap);
    obj.forEach((value, key) => {
      cloneMap.set(deepClone(key, hash), deepClone(value, hash));
    });
    return cloneMap;
  }

  // 处理 Set（注意：Set的值也是不可重复的）
  if (obj instanceof Set) {
    const cloneSet = new Set();
    hash.set(obj, cloneSet);
    obj.forEach((value) => {
      cloneSet.add(deepClone(value, hash));
    });
    return cloneSet;
  }

  // 处理数组
  if (Array.isArray(obj)) {
    const cloneArr = [];
    hash.set(obj, cloneArr);
    obj.forEach((item, index) => {
      cloneArr[index] = deepClone(item, hash);
    });
    return cloneArr;
  }

  // 处理普通对象
  const cloneObj = {};
  hash.set(obj, cloneObj);

  // 遍历可枚举的自身属性（不包含原型链上的）
  for (const key in obj) {
    if (Object.prototype.hasOwnProperty.call(obj, key)) {
      cloneObj[key] = deepClone(obj[key], hash);
    }
  }

  // 处理 Symbol 作为键的情况
  Object.getOwnPropertySymbols(obj).forEach((sym) => {
    cloneObj[sym] = deepClone(obj[sym], hash);
  });

  return cloneObj;
}
```

### 核心设计思路拆解

#### 1. 分支处理

函数开头用 `if (obj === null || typeof obj !== 'object')` 判断——这一行处理了原始类型（string、number、boolean、undefined、symbol、bigint）。这些类型不需要"深拷贝"，直接返回即可。

#### 2. 循环引用的处理（WeakMap 的妙用）

这是本实现中最关键的部分：

```javascript
if (hash.has(obj)) {
  return hash.get(obj);
}
// ...
hash.set(obj, cloneObj);
```

当第一次遇到对象 `obj` 时，将 `obj → cloneObj` 的映射存入 `hash`（一个 WeakMap）。如果后续再次遇到同一个 `obj`（即形成了循环引用），直接从 `hash` 中取出之前创建的副本并返回，避免无限递归。

**为什么用 WeakMap 而不是 Map？**

- `Map` 会持有对象的引用，导致对象无法被垃圾回收，引起内存泄漏。
- `WeakMap` 的键是弱引用的，不阻止垃圾回收，更适合处理引用关系。
- 只要克隆对象存在 WeakMap 中的键就不会被 GC，克隆完成后 WeakMap 自动清理。

#### 3. 特殊对象的处理

每种特殊类型都有独立的处理分支：

- **Date**：用 `new Date(obj.getTime())` 创建副本
- **RegExp**：保留 `source`（正则文本）和 `flags`（标志位）
- **Map**：遍历每个键值对，递归克隆值和键（键也可能是对象）
- **Set**：遍历每个元素，递归克隆后加入新 Set

#### 4. 原型链的处理

使用 `Object.prototype.hasOwnProperty.call(obj, key)` 判断属性是否为对象自身的属性，而不是从原型链上继承来的。这确保了深拷贝只拷贝自有属性，不会把原型上的方法或属性也复制过来。

#### 5. Symbol 键的处理

ES6 之后，对象的键可以是 Symbol，而 `for...in` 无法遍历 Symbol 键，必须用 `Object.getOwnPropertySymbols()` 单独处理。

#### 6. 稀疏数组的处理

```javascript
// 如果数组是稀疏的（含有空位），forEach 不会遍历空位
const sparse = [1, , 3]; // 稀疏数组，中间是空的
```

对于稀疏数组，上述实现中 `forEach` 不会遍历空位，空位会丢失。如果需要完整保留稀疏结构，需要额外判断：

```javascript
if (Array.isArray(obj)) {
  const cloneArr = [];
  hash.set(obj, cloneArr);
  obj.forEach((item, index) => {
    cloneObj[index] = deepClone(item, hash);
  });
  // 补充：处理稀疏数组的空位
  Object.keys(obj).forEach((key) => {
    if (!(key in cloneArr)) {
      cloneArr[key] = undefined;
    }
  });
  return cloneArr;
}
```

## 四、简化版深拷贝（面试快速写法）

如果面试时时间有限，可以先写一个简化版，涵盖最核心的功能：

```javascript
function deepClone(obj) {
  if (typeof obj !== 'object' || obj === null) {
    return obj;
  }

  const clone = Array.isArray(obj) ? [] : {};
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key]);
    }
  }
  return clone;
}
```

这个简化版能够处理大部分日常场景（嵌套对象和数组），但**不处理循环引用**、**不处理 Date/RegExp/Function/Symbol/Map/Set**。面试时可以先写简化版，再补充："实际工程中还需要处理循环引用（WeakMap）、特殊对象类型（Date/RegExp/Map/Set）、Symbol 键等。"

## 五、常见面试问题

### 1. 深拷贝和浅拷贝的区别是什么？

**浅拷贝**只拷贝对象的第一层属性。如果属性值是引用类型，拷贝的只是引用地址，修改副本仍会影响原对象。常见浅拷贝方式：`Object.assign()`、`{...obj}` 展开运算符、`Array.prototype.concat()` 和 `Array.prototype.slice()`。

**深拷贝**递归拷贝所有层级的属性，直到基本类型为止。修改深拷贝的副本完全不会影响原对象。

### 2. WeakMap 解决循环引用的原理是什么？

WeakMap 维护了一个 `obj → clone` 的映射表。当遇到已经处理过的对象时，直接从表中取出已有的克隆结果，避免了同一个对象被重复克隆导致的无限递归。这个映射表用 WeakMap 实现，是因为它的键是弱引用的——当克隆对象被垃圾回收时，对应的映射关系自动清除，不会造成额外内存泄漏。

### 3. 如果不用 WeakMap，还有什么方法解决循环引用？

可以用一个普通数组存储 `[原始对象, 克隆对象]` 的键值对，或者用 `Map`（但需要注意 Map 持有引用可能导致内存泄漏）。另一种方案是在拷贝前先遍历对象，检测并记录所有循环引用的路径，然后用一个全局的"已访问对象表"来去重。

### 4. 函数类型需要深拷贝吗？

通常**不需要**。函数在深拷贝场景中一般没有实际意义——拷贝函数意味着重新创建函数体，但函数体通常包含对外部作用域的引用（闭包），直接复制函数引用往往更安全。如果确实需要拷贝函数，可以用 `eval(func.toString())` 或 `new Function('return ' + func.toString())()` 的方式重建，但这会丢失闭包绑定。

### 5. JSON.parse(JSON.stringify()) 适合在什么场景使用？

适合拷贝**纯数据对象**——只包含基本类型、嵌套的基本类型和普通数组/对象，且不包含循环引用。典型的应用场景是：服务端返回的配置对象、简单的状态快照、不包含方法或特殊类型的纯数据对象。

---

## 六、总结

深拷贝的核心在于三点：

1. **递归**：逐层深入，直到遇见基本类型
2. **类型识别**：根据对象类型的不同，采取不同的克隆策略
3. **循环引用处理**：用 WeakMap 维护"已克隆对象"的映射表，防止无限递归

掌握了这些核心原理，面对任何面试中的深拷贝变体题（拷贝特定类型、处理环图、保持原型链等），都能从容应对。
