# 防抖与节流

防抖和节流是性能优化的重要技术，用于控制高频事件的触发频率。

## 1. 防抖 (Debounce)

防抖：事件触发 n 秒后才执行，如果 n 秒内再次触发，则重新计时。

```javascript
function debounce(fn, delay, immediate = false) {
  let timer = null
  
  return function(...args) {
    // 立即执行模式
    if (immediate && !timer) {
      fn.apply(this, args)
    }
    
    clearTimeout(timer)
    
    timer = setTimeout(() => {
      if (!immediate) {
        fn.apply(this, args)
      }
      timer = null
    }, delay)
  }
}

// 使用
const debouncedSearch = debounce(search, 300)

input.addEventListener('input', (e) => {
  debouncedSearch(e.target.value)
})
```

### 应用场景

- 搜索框输入（用户停止输入后才搜索）
- 窗口调整大小
- 表单验证

## 2. 节流 (Throttle)

节流：n 秒内只执行一次，稀释执行频率。

```javascript
function throttle(fn, delay) {
  let lastTime = 0
  let timer = null
  
  return function(...args) {
    const now = Date.now()
    
    // 时间戳版本
    if (now - lastTime >= delay) {
      fn.apply(this, args)
      lastTime = now
    }
    
    // 定时器版本（保证最后一次执行）
    if (!timer) {
      timer = setTimeout(() => {
        fn.apply(this, args)
        timer = null
      }, delay)
    }
  }
}

// 使用
const throttledScroll = throttle(handleScroll, 100)

window.addEventListener('scroll', () => {
  throttledScroll()
})
```

### 应用场景

- 滚动事件
- 鼠标移动
- 按钮点击（防止重复提交）

## 3. 防抖 vs 节流

| 特性 | 防抖 | 节流 |
|------|------|------|
| 执行时机 | 事件停止后 | 固定间隔 |
| 首次触发 | 可配置立即执行 | 等待间隔后 |
| 最后一次 | 保证执行 | 可能丢失 |
| 适用场景 | 输入类事件 | 状态同步类事件 |

## 4. leading 和 trailing 选项

```javascript
function debounce(fn, delay, options = {}) {
  const { leading = false, trailing = true } = options
  let timer = null
  
  return function(...args) {
    const invokeImmediately = leading && !timer
    
    if (timer) clearTimeout(timer)
    
    timer = setTimeout(() => {
      if (trailing) fn.apply(this, args)
      timer = null
    }, delay)
    
    if (invokeImmediately) fn.apply(this, args)
  }
}
```

## 5. 取消功能

```javascript
function debounce(fn, delay) {
  let timer = null
  
  const debounced = function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
      timer = null
    }, delay)
  }
  
  debounced.cancel = () => {
    clearTimeout(timer)
    timer = null
  }
  
  return debounced
}

// 使用
const debouncedFn = debounce(fn, 300)
debouncedFn()
debouncedFn.cancel() // 取消
```

## 6. 面试高频问题

### Q: 防抖和节流的区别？

- 防抖：事件频繁触发后，只执行最后一次
- 节流：事件频繁触发，每隔一段时间执行一次

### Q: 各自的应用场景？

- 防抖：搜索框、表单验证
- 节流：滚动、鼠标移动、resize

### Q: 如何实现带取消的防抖？

保存定时器 ID，提供 cancel 方法清除定时器。

## 7. 总结

防抖和节流是控制函数执行频率的重要技巧，选择哪个取决于业务场景：
- 需要最终结果 → 防抖
- 需要平滑更新 → 节流
