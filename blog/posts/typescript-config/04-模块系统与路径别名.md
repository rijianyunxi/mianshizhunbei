# 模块系统与路径别名

## moduleResolution 的选择

| 值 | 适用场景 | 导入写法 |
|----|---------|---------|
| `node16` / `nodenext` | 严格 Node ESM | 必须写 `./utils/index.js` |
| `bundler` | tsx / tsup 项目 | 省略后缀 `./utils/index` |
| `node` | 老项目 | 省略后缀 `./utils/index` |

用 `tsx` 跑的项目推荐 `bundler`，省掉后缀烦恼：

```json
{
  "module": "esnext",
  "moduleResolution": "bundler"
}
```

## 允许写 .ts 后缀

默认 TS 不允许导入时写 `.ts` 后缀，开启这个选项：

```json
{
  "allowImportingTsExtensions": true,
  "noEmit": true  // 必须同时开启
}
```

```typescript
import { helper } from "./utils/helper.ts"  // ✅
```

## 路径别名

TS 6.0 起 `baseUrl` 已弃用，直接用 `paths`：

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"],
      "@utils/*": ["./src/utils/*"],
      "@types/*": ["./src/types/*"]
    }
  }
}
```

```typescript
// 使用别名
import { helper } from "@/utils/helper"
import type { User } from "@types/user"
```

## tsx 自动支持别名

`tsx` 4.x 会自动读取 `tsconfig.json` 的 `paths`，不需要额外配置，直接跑就行：

```bash
npx tsx src/index.ts  # 自动识别 @ 别名
```

## package.json type 字段

```json
{
  "type": "module"
}
```

加上这个，Node.js 会把 `.js` 文件当 ESM 处理（支持 `import/export`）。

不加则默认 CommonJS（`require/module.exports`）。

用 `tsx` + `module: esnext` 的项目建议加上。
