# 虚拟滚动实现

虚拟滚动是处理大数据列表的利器，通过只渲染可视区域内的元素，大幅提升性能。

## 1. 为什么需要虚拟滚动？

假设有 10000 条数据：
- 普通渲染：创建 10000 个 DOM 节点 → 卡顿
- 虚拟滚动：只创建可视区域的 ~20 个节点 → 流畅

## 2. 核心原理

```
可视区域（viewport）
┌─────────────────┐
│ ┌─────────────┐ │
│ │ Item 50     │ │ ← 渲染
│ │ Item 51     │ │
│ │ Item 52     │ │
│ │ Item 53     │ │
│ │ Item 54     │ │
│ └─────────────┘ │
│                 │
└─────────────────┘

实际 DOM 高度：10000 × itemHeight
滚动容器高度：viewportHeight
```

## 3. 固定高度虚拟滚动

```javascript
class VirtualList {
  constructor(options) {
    this.container = options.container
    this.itemHeight = options.itemHeight || 50
    this.items = options.items
    this.buffer = options.buffer || 3
    
    this.init()
  }
  
  init() {
    // 计算总高度（用 spacer 占位）
    this.spacer = document.createElement('div')
    this.spacer.style.height = `${this.items.length * this.itemHeight}px`
    this.container.appendChild(this.spacer)
    
    // 列表容器
    this.listContainer = document.createElement('div')
    this.listContainer.style.position = 'relative'
    this.container.appendChild(this.listContainer)
    
    // 绑定滚动事件
    this.container.addEventListener('scroll', () => this.onScroll())
    
    this.render()
  }
  
  onScroll() {
    requestAnimationFrame(() => this.render())
  }
  
  render() {
    const scrollTop = this.container.scrollTop
    const viewportHeight = this.container.clientHeight
    
    // 计算可见范围
    const startIndex = Math.floor(scrollTop / this.itemHeight)
    const endIndex = Math.ceil((scrollTop + viewportHeight) / this.itemHeight)
    
    // 缓冲区域
    const start = Math.max(0, startIndex - this.buffer)
    const end = Math.min(this.items.length, endIndex + this.buffer)
    
    // 清空并重新渲染
    this.listContainer.innerHTML = ''
    
    for (let i = start; i < end; i++) {
      const item = document.createElement('div')
      item.className = 'virtual-item'
      item.style.position = 'absolute'
      item.style.top = `${i * this.itemHeight}px`
      item.style.height = `${this.itemHeight}px`
      item.textContent = this.items[i]
      this.listContainer.appendChild(item)
    }
  }
}
```

## 4. transform 优化

使用 transform 代替 top，避免重排：

```javascript
// ❌ 差：每次修改 top 都触发重排
item.style.top = `${i * itemHeight}px`

// ✅ 好：transform 不触发重排
item.style.transform = `translateY(${i * itemHeight}px)`
```

## 6. 面试高频问题

### Q: 虚拟滚动如何处理动态高度？

1. 预估高度初始化
2. 渲染时测量实际高度
3. 缓存测量结果
4. 动态调整滚动条

### Q: 为什么用 transform 而不是 top？

transform 不触发重排重绘，只触发合成，性能更好。

### Q: transform 对 fixed 定位的影响？

transform 会创建新的层叠上下文，导致子元素的 fixed 定位相对于 transform 祖先而不是视口。

## 7. 总结

虚拟滚动的核心：
1. 计算可视区域索引
2. 只渲染可视区域元素
3. 用 transform 定位避免重排
4. 动态高度需要测量和缓存
